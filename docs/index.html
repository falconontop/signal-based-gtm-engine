Scope
When NOT to use this system
  • Low-ticket, high-volume B2C — signal cost outweighs value per lead
  • Markets without clear digital signals — no job posts, no funding data, no traceable intent
  • Early-stage products without a defined ICP — scoring logic needs a target to score against
  • Teams unwilling to do the 15-min review step — garbage in, garbage out at sending stage

03
Agent — Core Logic
Fit + Intent Scoring
Combines enrichment data into a 0–100 score. Filters to qualified leads only. The quality gate of the whole system.
Signals the agent detects
Per-company research
  • Job postings (SDR / AE / RevOps / GTM)
  • LinkedIn company headcount growth
  • Recent funding rounds (Crunchbase)
  • Tech stack signals (BuiltWith, job post keywords)
  • Content activity (posting frequency, ads running)
Code pattern
async detectSignals(company) { const jobs = await searchJobs(company) const funding = await checkFunding(company) const stack = await detectStack(company) return { hiringSignal: jobs.length > 0, fundingSignal: funding.recent, stackSignal: stack.hasNewCRM, rawJobs: jobs // used for email hook } }
Output
Signal object
  • Structured JSON per company
  • Specific job titles found (used in email hook)
  • Funding amount + date if available
  • Feeds directly into scoring engine

Scoring logic
Fit score (0–100)
  • Company size 10–200 → +20
  • Industry: SaaS / Tech → +20
  • Hiring for sales roles → +15
  • Has CRM (HubSpot/SF) → +15
  • Subscription / high ACV → +10
  • Threshold: ≥60 passes through
Intent scoring
Intent score (0–100)
  • Hiring signal → +30
  • Funding / growth → +25
  • Website activity → +20
  • Tech stack change → +15
  • Threshold: ≥40 passes through
  • Both thresholds must pass → MQL
Code pattern
function scoreLead(lead, signals) { let fit = 0, intent = 0 // fit if (inRange(lead.size,10,200)) fit+=20 if (lead.industry==='SaaS') fit+=20 if (signals.hiringSignal) fit+=15 // intent if (signals.hiringSignal) intent+=30 if (signals.fundingSignal) intent+=25 return { fit, intent, qualified: fit>=60 && intent>=40 } }

Agent — Intelligence
Signal Brief
Per qualified lead: synthesizes signals into a structured brief. The context that makes personalization real, not fake.
05
Agent — Generation
4-Touch Email Sequence Generation
Uses signal brief + ICP angle + your messaging framework to write Day 1, 3, 6, 10 emails. Structured output. AI-assisted but grounded in actual company signals — not generic copy.
What's in a signal brief
Structured per-lead context
  • Company name, size, industry, HQ
  • Key signal: "Hiring 2 SDRs + 1 RevOps"
  • Hook angle: "scaling outbound without playbook"
  • Funding context if relevant
  • Existing tools (CRM, SEP detected)
  • Decision maker name + title
Why this step matters
The personalization root
  • Email gen references brief — not raw data
  • Keeps generation prompt clean and short
  • Brief is reviewable by human before sending
  • Reusable across channels (email + LinkedIn)
Output format
{ "company": "Acme SaaS GmbH", "signal": "hiring 2 SDRs + RevOps", "hook": "scaling outbound team", "angle": "no playbook yet", "contact": "Jan Müller, VP Sales", "crm": "HubSpot" }
The 4 touches
Sequence structure
  • Day 1 — Signal hook: "Saw you're hiring SDRs..."
  • Day 3 — Value add: relevant insight / stat
  • Day 6 — Case angle: proof for their exact situation
  • Day 10 — Break-up: concise, no hard sell
System prompt design
What makes it not generic
  • Your voice + tone baked into system prompt
  • 3 example sequences (few-shot) in prompt
  • Signal brief passed as structured context
  • AI-assisted but structured — few-shot examples enforce quality floor
  • Word count enforced per touch
This is 40% of the real work
Prompt engineering note
  • Quality = quality of your system prompt
  • Needs real example emails from past wins
  • Needs explicit anti-patterns ("never say synergy")
  • Iterate on 20 leads before full batch

System limitations
Where this breaks — and what to do about it
Signal layer
Signal noise in early batches
Job board data is laggy. First 2–3 runs need manual brief validation before trusting automation. Calibrate signal weights on real reply data, not assumptions.
Email layer
Deliverability is not the agent's job
Great copy doesn't matter if the domain isn't warmed up. Agent hands off to Instantly — but domain setup, DMARC, SPF, and warm-up are prerequisites the agent can't fix.
Cost layer
Clay gets expensive at scale
Clay's waterfall enrichment costs rise linearly with volume. For <100 leads/week, agent-only research is viable. Above that, Clay ROI is positive — but it's a real line item.
Generation layer
AI-assisted copy still needs human review (~10–15%)
Hallucinated context, wrong seniority, tone mismatch — rare but real. The human review step at export isn't optional. 15 min per 200 leads is the actual cost of running this safely.
